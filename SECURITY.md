# Security and editorial concerns

This repository is an editorial list, not a software product, so "security" here covers two things:

## 1. Concerns about a listed tool

If you believe a project listed here is unsafe, abandoned, compromised, has been acquired by an actor whose use of it conflicts with the public-interest framing of this list, or otherwise should no longer carry our endorsement — please report it privately rather than opening a public issue.

The fastest channel is a [private security advisory](https://github.com/HappySoftwareLabs/awesome-civic-ai/security/advisories/new) on this repository. We will review and, if the concern is borne out, remove or update the entry.

You can also open a public issue using the "Stale entry" template if the concern is purely about maintenance (no commits in 12+ months, archived repo) and not about safety or trust.

## 2. Code of Conduct violations

Reports about harassment or other Code of Conduct violations should also go through a [private security advisory](https://github.com/HappySoftwareLabs/awesome-civic-ai/security/advisories/new). Reporter identity is kept confidential.

## What this file does *not* cover

There is no software in this repository to find vulnerabilities in — it is a curated README with supporting documents. Reports about vulnerabilities in a *listed project* should go to that project's own security contact, not to us.
